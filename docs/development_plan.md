# План разработки NeuroSlicer (Python)

## 1. Источники данных
1. ✅ Подключить troubleshooting guide как основной источник markdown-кейсов (autodetect + env path).
2. Реализовать преобразование markdown -> нормализованные записи (`category`, `symptoms`, `causes`, `recommendations`).
3. Добавить валидацию качества и покрытие категорий дефектов.

## 2. Диагностический контур
1. Базовый retrieval по базе кейсов.
2. AI-слой через Hugging Face (`openai/gpt-oss-120b`) для ранжирования и объяснений.
3. Возврат структурированного результата: тип дефекта, причины, рекомендации, confidence, источники.

## 3. Интеграция с профилями слайсера
1. ✅ Базовая поддержка профилей OrcaSlicer (JSON/INI).
2. ✅ Маппинг рекомендаций на diff-патч параметров.
3. ✅ Добавлен безопасный предпросмотр через `--dry-run` (без записи файла).
4. ⏳ Следующий шаг: интерактивное подтверждение пользователя перед записью изменений.

## 4. Качество
1. ✅ Unit-тесты парсинга, диагностики и profile diff.
2. ⏳ Контрактные тесты формата LLM-ответов.
3. ⏳ Benchmark-метрики времени ответа.
